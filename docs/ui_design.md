# UI Design

## UI方針
- Web UI (Next.js App Router) を採用
- PoC向けに最小構成
- ユーザーが各フェーズの進行を理解できることを優先

## 画面構成 (Web UI/Next.js)
1. テーマ入力画面 `/`
   - テキスト入力
   - 開始ボタン
2. 議論・質問画面 `/projects/[id]`
   - 質問表示
   - 回答入力
   - 進捗表示
3. リサーチ結果画面 `/projects/[id]`
   - 収集情報の一覧
   - Markdown保存ボタン
4. 構成案画面 `/projects/[id]`
   - 構成案プレビュー
   - 修正入力
5. 原稿画面 `/projects/[id]`
   - 原稿プレビュー
   - 修正入力
6. 完了画面 `/projects/[id]`
   - 完成原稿のMarkdownダウンロード

## UI要件
- 状態ステップが常に見えること
- 編集可能箇所は明確にすること
- 成果物はMarkdownとして保存できること
- 成果物はプレビュー表示できること（Markdownレンダリング）
- 生成済み成果物のみダウンロードリンクを表示すること
- 現在ステータスに対応する成果物を強調表示すること
- Markdownプレビューは以下の要素を視認できること:
  - 見出し（H1〜H4）
  - 段落、リスト
  - 引用（blockquote）
  - コードブロック/インラインコード
  - テーブル
  - リンク
- 成果物一覧は「すべて展開 / すべて折りたたむ」で一括操作できること
- 成果物フィルタの状態はURL（`?type=`）に反映し、履歴に残すこと
- フィルタ未指定の場合は `localStorage` の前回設定を復元すること
- 成果物の操作は三点メニューに集約し、クリック外/ESCで閉じること
- 成果物メニューはフォーカスリングを表示し、キーボード操作を阻害しないこと
- 成果物操作メニューには以下の操作があること:
  - 種類フィルタ
  - すべて展開 / すべて折りたたむ
  - 一覧を更新

## スタイルガイド (暫定)
- 余白を多めにし、段階が分かるレイアウト
- ステップ表示は水平 or 縦ステップで明確に
