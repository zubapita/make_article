# Requirements

## プロジェクト概要
- プロジェクト名: make_article
- 種別: 実験用PoC
- 目的: テーマ入力から議論・リサーチを経て、構成案・原稿を作成し、Markdownで保存する。

## 主要ゴール
1. テーマや疑問点を入力すると、対話しながらWebリサーチを実行できる。
2. 議論結果・リサーチ結果をMarkdownで保存できる。
3. 構成案を作成し、合意の上で原稿を生成できる。
4. 記事企画から原稿完成までのワークフローを滞りなく完結できる。
5. Web UIまたはCLI UIが、既存ツール（Web/Claude Desktop/Claude Code）より便利だと感じられる。

## 非機能要件
- 既存のCLI型AIエージェント（Claude Code / Codex CLI / Gemini CLI / OpenCode など）をヘッドレス運用できること。
- AIエージェントのSKILLは標準化された形式で保存し、呼び出し可能であること。
- 各記事プロジェクトは指定ワーキングディレクトリ内で完結して保存されること。
- 実験用途のため、最小実装で動作検証が可能なこと。
- MarkdownプレビューはサニタイズされたHTMLで表示し、XSSを防止すること。

## スコープ
### やること
- 記事企画から原稿完成までのフローを実装する。
- 各記事ごとに作業ディレクトリを生成し、成果物を保存する。
- 一般的に「便利」と感じられるUI/UXの最低限を提供する。

### やらないこと
- 要求仕様にない機能の追加。
- 高度な最適化や過剰な抽象化。
- スケールやマルチユーザー対応の本格運用。

## ユースケース
1. ユーザーがテーマを入力する。
2. システムが議論を開始し、必要な質問を提示する。
3. システムがWebリサーチを実行し、結果を保存する。
4. システムが構成案を作成し、ユーザーと合意形成する。
5. システムが記事原稿を生成し、再度議論・修正する。
6. 完成原稿をMarkdownで保存する。

## 成功基準
- 企画から原稿完成までの一連のフローが中断せずに完了する。
- UI/UXが既存ツールよりも便利と判断される。

## 前提・制約
- 各CLIエージェントは実行可能で、必要なMCP/Skillが設定済みである。
- Webリサーチには外部アクセスが必要。
- PoCのためデータはローカル保存で十分とする。
